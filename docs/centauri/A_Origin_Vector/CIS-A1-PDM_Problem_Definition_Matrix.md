## 1. Purpose
Define the mission need, operational constraints, and lunar relevance for the candidate technology. Establish the foundational problem statement that drives all downstream Centauri phases.

## 2. Inputs
- Artemis mission objectives
- ConOps documentation
- Known capability gaps
- Environmental constraints (thermal, vacuum, dust, radiation)

## 3. Outputs
- Mission Need Statement (MNS)
- Constraint matrix
- Initial success criteria
- Problem framing narrative

## 4. Method
1. Extract mission drivers from ConOps.
2. Identify explicit and implicit capability gaps.
3. Translate gaps into structured problem statements.
4. Validate with cross-discipline review.

## 5. Mission Integration
- Maps to Artemis surface operations
- Identifies EVA, logistics, and comms dependencies
- Establishes early integration hooks

## 6. TRL & SBIR Mapping
- TRL: 1
- SBIR: Pre-Phase I ideation

## 7. Risks & Mitigations
- **Risk:** Misalignment with evolving mission priorities
  **Mitigation:** Refresh at each major review
- **Risk:** Overly narrow framing
  **Mitigation:** Include multi-domain review
